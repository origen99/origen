# Guia de contribucion (Issue-first)

## Objetivo
Esta guia permite proponer mejoras de contenido sin necesidad de editar markdown directamente en la primera fase.

## Flujo rapido
1. Abrir un Issue de propuesta documental.
2. Describir el problema que quieres resolver.
3. Explicar el argumento principal.
4. Añadir evidencia, ejemplo o fuente.
5. Indicar el tipo de contenido y alcance.

## Resumen en 30 segundos
- Abre una propuesta con el template de Issue.
- Define problema, argumento y evidencia minima.
- Selecciona tipo de aporte y alcance.
- Espera triaje editorial y feedback de aceptacion o mejora.

## Campos minimos del Issue
- `title`: resumen breve de la propuesta
- `problem_statement`: que problema aborda
- `argument_summary`: razonamiento principal
- `evidence_or_sources`: evidencia, ejemplo o fuente
- `content_type`: new-doc, edit-doc o correction
- `scope`: home, tutorials, how-to, reference o explanation

## Tono editorial
- Escribe para invitar a pensar, no para imponer.
- Evita etiquetas identitarias como herramienta de persuasion.
- Prioriza claridad, evidencia y razonamiento.

## Nota de fase
En esta V1 el canal principal de colaboracion es Issue-first. La edicion directa por PR se ampliara en fases posteriores.
